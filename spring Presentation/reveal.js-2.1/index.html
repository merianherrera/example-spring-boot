Historia De Spring

Spring es un framework para el desarrollo de aplicaciones y contenedor de inversión de control, de código abierto para la plataforma Java.

La primera versión fue escrita por Rod Johnson.

El framework fue lanzado inicialmente bajo la licencia Apache 2.0 en junio de 2003. El primer gran lanzamiento fue la versión 1.0, que apareció en marzo de 2004 y fue seguida por otros hitos en septiembre de 2004 y marzo de 2005.

Historia De java

Es un lenguaje de programación orientado a objetos que fue diseñado específicamente para tener tan pocas dependencias de implementación como fuera posible.

Pretente que los desarrolladores escriban el programa una sola vez a esto se le conoce como WORA que quiere decir que el programa ya escrito una vez puede correr en cualquier plataforma no necesita ser recompilado.

Desarrollado originalmente por James Gosling de Sun Microsystems (adquirida por Oracle).

Su sintaxis deriva principalmente de C y C++ pero con menos utilidades a bajo nivel que ellos.

Ventajas del framework

-Facil de configurarlo

-Integracion con otras herramientas

-Codigo ordenado

-Uso de anotaciones

-Facil creacion de objetos

-Facil programacion de formularios

-Estandar de programacion

Desventajas del framework

La configuración de Spring está inflada, es decir, para cada servicio que se tenga hemos de configurarlo en un XML de configuración. Aunque hay otras formas de configuración de Spring aparte del XML puro: programando por medio de la API, mediante un estándar JSR y con un mínimo XML y anotaciones.

No se puede evaluar si un objeto ha sido bien inyectado más que en tiempo de ejecución. Aunque hay herramientas como Spring IDE que sí que ayudan.

El contenedor de Spring no es ligero no es recomendable su uso en aplicaciones de tiempo real o en aplicaciones para móviles.
